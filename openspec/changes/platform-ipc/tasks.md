## 1b. Per-user IPC runtime directory rooted at an un-squattable base (Decision 6 — root-cause security fix)

- [ ] 1b.1 Add a SINGLE shared resolver (used by GUI bind, MCP shim connect, AND the separate `hooks/cli.rs` binary) keyed off **`getuid()`**, NOT a process-local env var. Root per platform: **Linux** = `/run/user/<getuid()>/nergal/` (un-writable by other uids); NO `$XDG_RUNTIME_DIR` override at all (a weaker-than-string-equal override reopens the divergence for non-standard `$XDG_RUNTIME_DIR`; a string-equal one is a no-op — omit it). If `/run/user/<uid>` is missing/invalid, fall back to a `0700` dir under `getpwuid(getuid())->pw_dir` (NOT `$HOME` — `sudo`/`su` leaves `$HOME` at the original user), validating the fallback parent is owned by `getuid()`; NEVER a guessable-named dir under shared `/tmp`. **macOS** = `temp_dir()/nergal/` (already per-user `0700`). Create `0700` if absent; if it exists, validate owned-by-current-uid and not-broader-than-`0700`; on a foreign-owned/over-permissive dir, refuse the affected IPC surface and log (tripwire — unreachable by another uid given the un-squattable root). WHY `getuid()` not `$XDG_RUNTIME_DIR`: Codex sanitizes the MCP shim env (see project memory; the shim recovers `NERGAL_SESSION_ID` from `/proc/<pid>/environ`), so an env-keyed root would make the shim's connect path diverge from the GUI's bind path and silently break MCP.
- [ ] 1b.2 The resolver also owns the `AF_UNIX` `sun_path` overflow check + deterministic short-hash fallback (§1.1), so all three processes derive identical paths; no per-call-site reimplementation.
- [ ] 1b.3 Repoint EVERY endpoint path through the resolver: hook socket (`config.rs:305`), MCP socket (`mcp/mod.rs:46`), plan-review FIFO (`hooks/cli.rs:142`), ask-user FIFO (`agents/claude_code/adapter.rs`). Keep the directory name and endpoint filenames short (macOS `sun_path` budget; the `/run/user/<uid>` root is short, the home fallback relies on the hash backstop).
- [ ] 1b.4 Guard concurrent same-uid instances: an `flock`'d lockfile in the per-user dir + probe-then-adopt-or-fail. Liveness predicate = `connect()` success (an instance is live → defer) vs `ECONNREFUSED` on an existing socket file (stale, no listener → provably dead → unlink), NOT a reply (the hook socket is fire-and-forget and never answers). Stale-endpoint removal is conditional on this probe, never unconditional.
- [ ] 1b.5 Handle the `/run/user/<uid>` teardown edge: if the IPC dir vanishes mid-run (systemd removes it on last-session logout), the server **re-runs the full resolver** (NOT `mkdir /run/user/<uid>` — non-root gets `EACCES` post-teardown; the resolver falls to the home fallback) rather than dying silently. Accept + document the churn-window divergence (GUI on fallback while post-login children compute `/run/user/<uid>`).
- [ ] 1b.6 Verify: on Linux, endpoints resolve under `/run/user/<uid>/nergal/` AND the Codex MCP shim (sanitized env) connects to the same path the GUI bound; a foreign-owned dir yields a logged refusal; a second same-uid launch does not unlink the live instance's socket; full check green.

## 4. Hook CLI onto the seam + blocking-FIFO hardening

- [ ] 4.1 `hooks/cli.rs`: remove the top-level `use std::os::unix::net::UnixStream;` (`:3`); route the five connect sites (`:16,47,178,250,292`) through the seam's sync `PlatformStream::connect`.
- [ ] 4.2 Keep the plan-review (`:142,146,188`) and ask-user (`agents/claude_code/adapter.rs` register/`submit_ask_answer`) round-trips on POSIX FIFOs for Linux+macOS (the FIFO→`PlatformStream` unification is sequenced with Windows), BUT apply the three hardenings that do not wait (design Decision 5): (a) the FIFO lives inside the per-user IPC dir from §1b so only a same-uid process can write the decision — closes the forged-`allow` approval-gate hole; (b) make the blocking read **liveness-aware** with FIFO-real primitives (NOT a connection-close signal — the FIFO has no connected writer during deliberation): the GUI writes `gui.pid` (pid AND a process-start-time token — Linux `/proc/<pid>/stat` starttime, macOS `proc_pidinfo`; `pidfd` preferred on Linux) into the per-user dir; the hook CLI opens the FIFO non-blocking (`O_RDONLY|O_NONBLOCK`) and `poll()`s with an explicit bounded timeout (≈1 s, NOT `poll(-1)`/POLLHUP-edge — a writerless FIFO returns `POLLHUP` continuously with platform-divergent semantics → busy-spin/stall), ignoring `POLLHUP`, reading on `POLLIN`, and on each timeout tick checking liveness via pid+start-time token — `ESRCH` OR start-time mismatch (pid recycled) resolves to a safe deny so the agent loop neither hangs forever nor silently degrades fast-detection to the backstop, but a live GUI (pid+token match) with a pending human decision keeps waiting (a wall-clock force-deny mid-deliberation is a regression); any wall-clock backstop is human-scale + configurable and surfaces a re-arm rather than a silent deny; (c) unlink the FIFO on entry and on exit (RAII guard) and treat a pre-existing FIFO at the path as hostile (remove, do not reuse) so a stale/pre-seeded decision from a crashed run or recycled PID is never read as live. Add a code comment pointing at the deferred full unification (WHY).
