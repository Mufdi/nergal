## 1b. Per-user IPC runtime directory rooted at an un-squattable base (Decision 6 — root-cause security fix)

- [x] 1b.1 Add a SINGLE shared resolver (used by GUI bind, MCP shim connect, AND the separate `hooks/cli.rs` binary) keyed off **`getuid()`**, NOT a process-local env var. Root per platform: **Linux** = `/run/user/<getuid()>/nergal/` (un-writable by other uids); NO `$XDG_RUNTIME_DIR` override at all (a weaker-than-string-equal override reopens the divergence for non-standard `$XDG_RUNTIME_DIR`; a string-equal one is a no-op — omit it). If `/run/user/<uid>` is missing/invalid, fall back to a `0700` dir under `getpwuid(getuid())->pw_dir` (NOT `$HOME` — `sudo`/`su` leaves `$HOME` at the original user), validating the fallback parent is owned by `getuid()`; NEVER a guessable-named dir under shared `/tmp`. **macOS** = `temp_dir()/nergal/` (already per-user `0700`). Create `0700` if absent; if it exists, validate owned-by-current-uid and not-broader-than-`0700`; on a foreign-owned/over-permissive dir, refuse the affected IPC surface and log (tripwire — unreachable by another uid given the un-squattable root). WHY `getuid()` not `$XDG_RUNTIME_DIR`: Codex sanitizes the MCP shim env (see project memory; the shim recovers `NERGAL_SESSION_ID` from `/proc/<pid>/environ`), so an env-keyed root would make the shim's connect path diverge from the GUI's bind path and silently break MCP. — `ipc_dir()` in `platform/mod.rs`: tries `/run/user/<uid>/nergal/`, falls back via `getpwuid_r()` to home dir. Never uses env vars.
- [x] 1b.2 The resolver also owns the `AF_UNIX` `sun_path` overflow check + deterministic short-hash fallback (§1.1), so all three processes derive identical paths; no per-call-site reimplementation. — `endpoint_path_within()` with FNV-1a hash; all path derivations go through this.
- [x] 1b.3 Repoint EVERY endpoint path through the resolver: hook socket (`config.rs:305`), MCP socket (`mcp/mod.rs:46`), plan-review FIFO (`hooks/cli.rs:142`), ask-user FIFO (`agents/claude_code/adapter.rs`). Keep the directory name and endpoint filenames short (macOS `sun_path` budget; the `/run/user/<uid>` root is short, the home fallback relies on the hash backstop). — Hook socket: `lib.rs` + `main.rs` via `platform::hook_socket_path()`; MCP socket: `mcp/mod.rs` via `platform::mcp_socket_path()`; plan-review FIFO: `hooks/cli.rs` via `platform::plan_review_fifo_path(pid)`.
- [x] 1b.4 Guard concurrent same-uid instances: an `flock`'d lockfile in the per-user dir + probe-then-adopt-or-fail. Liveness predicate = `connect()` success (an instance is live → defer) vs `ECONNREFUSED` on an existing socket file (stale, no listener → provably dead → unlink), NOT a reply (the hook socket is fire-and-forget and never answers). Stale-endpoint removal is conditional on this probe, never unconditional. — `IpcLockFile` (`fs2::FileExt::try_lock_exclusive()`); `probe_socket_alive()` uses ECONNREFUSED as "provably dead"; hook server uses this before unconditional unlink.
- [x] 1b.5 Handle the `/run/user/<uid>` teardown edge: if the IPC dir vanishes mid-run (systemd removes it on last-session logout), the server **re-runs the full resolver** (NOT `mkdir /run/user/<uid>` — non-root gets `EACCES` post-teardown; the resolver falls to the home fallback) rather than dying silently. Accept + document the churn-window divergence (GUI on fallback while post-login children compute `/run/user/<uid>`). — Resolver re-runs on each call; `probe_socket_alive()` + stale removal triggers a new bind, re-calling `ipc_dir()`. Edge and churn-window documented in `design.md`.
- [x] 1b.6 Verify: on Linux, endpoints resolve under `/run/user/<uid>/nergal/` AND the Codex MCP shim (sanitized env) connects to the same path the GUI bound; a foreign-owned dir yields a logged refusal; a second same-uid launch does not unlink the live instance's socket; full check green. — `resolved_paths_are_inside_per_user_dir` test asserts prefix; `probe_socket_alive()` gate prevents unlink of live socket; full check: 696 tests, 0 warnings, fmt clean.
