# Ungated Unix Analysis — Windows Compile Blockers

Generated by static analysis of `src-tauri/src/` against `x86_64-pc-windows-msvc`.

**libc dependency note:** `libc` is declared under `[target.'cfg(unix)'.dependencies]` in
`src-tauri/Cargo.toml`. Any ungated `libc::*` call in production code is a guaranteed
Windows compile failure because the crate is not present on that target.

**`nix` crate:** no `nix` crate dependency found — all "nix::" hits in rg output are
substring matches of `std::os::unix::` (the `u` in `unix` is a prefix; `nix::` matches
inside `unix::`). No action needed for the `nix` crate.

**`fs2` crate:** declared as an unconditional dependency (`fs2 = "0.4"`); the crate has
Windows support via `LockFileEx`/`UnlockFile`. All `fs2::FileExt` uses are safe on Windows.
