No SQLite schema changes. Win32 API surfaces + signatures are in `handoff/windows-namedpipe-research.md` (CONFIRMED/UNCONFIRMED marked there); this document maps them to the seam and an execution order. ## Verified codebase facts (do not re-assume)

Verified against current source 2026-06-28 (post `windows-compile` authoring; line refs are pre-#1-implementation and shift once #1 lands — re-anchor at implementation time):

- **Seam shape** (`platform/mod.rs`): `#[cfg(unix)] PlatformListener { listener: UnixListener, path }` with `bind(&Path) -> io::Result<Self>` (`:600`), `accept(&self) -> io::Result<(PlatformStream, u32)>` (`:647`, calls `peer_uid_of` `:649`), `path()`; `#[cfg(unix)] PlatformStream(UnixStream)` with `connect(&Path)` (`:674`), `inner()`, `AsyncRead`+`AsyncWrite` (`:689,700`). `#[cfg(windows)]` `Unsupported` stubs at `:730-807`. `peer_uid_of(&tokio::net::UnixStream) -> io::Result<u32>` `:497` (`peer_cred().uid()`). `sync_connect(&Path) -> io::Result<UnixStream>` `:820` (`#[cfg(unix)]`; `windows-compile` adds a `#[cfg(windows)] UnsupportedSyncStream` stub).
- **Framing helpers** are generic over `AsyncRead`/`AsyncWrite` (`mcp/transport.rs` `read_frame`/`write_frame`) — they operate over any stream, including `NamedPipeServer`/`NamedPipeClient` (both impl `AsyncRead + AsyncWrite`, research §1).
- **MCP transport**: `UnixSocketTransport { listener: UnixListener }` with `bind`/`accept` (returns `(UnixStream, u32)`)/`connect`/`peer_uid`. `windows-compile` gates it `#[cfg(unix)]`. The daemon spawn is `lib.rs:692-714` (`UnixSocketTransport::bind` + `libc::getuid()` app_uid), gated `#[cfg(unix)]` by #1.
- **Hook server**: `hooks/server.rs` binds raw `UnixListener` (`:222`), accept loop uses `peer_uid_of` (`:254`) + `probe_socket_alive` (`:206`). `windows-compile` gates the bind `#[cfg(unix)]`.
- **Blocking path**: `hooks/cli.rs` `plan_review` uses `#[cfg(unix)]` `FifoGuard` (mkfifo) + `blocking_fifo_read_liveness_aware` (`:316-396`, `RawFd`/`poll`); a broken `#[cfg(not(unix))]` placeholder at `:197-207` constructs `FifoGuard(fifo_path.clone())` whose `Drop` unlinks the FIFO immediately (runtime bug — flagged by windows-compile iprev; this change removes it). `sync_connect` callers: `cli.rs:15,46,235,457,509` (sync `Read`/`Write`).
- **Endpoint paths** (`platform/mod.rs:274-324`): `hook_socket_path`/`mcp_socket_path`/`plan_review_fifo_path`/`gui_pid_path` each have `#[cfg(unix)]` (ipc_dir join) + `#[cfg(not(unix))]` (temp_dir) branches — the `not(unix)` branches return placeholder filesystem paths and SHALL be replaced by pipe-name strings on Windows.
- **`windows` crate** is already a transitive dep on Windows (via Tauri/`tao`/`wry`); this change adds Win32 *features*, not a new vendor.

## Edit plan (execution order)

### Step 1 — Cargo: windows Win32 features
Add `[target.'cfg(windows)'.dependencies.windows]` with `features = ["Win32_Foundation", "Win32_Security", "Win32_Security_Authorization", "Win32_System_Memory", "Win32_System_Pipes", "Win32_System_Threading", "Win32_System_IO", "Win32_Storage_FileSystem"]` (version pinned to whatever Tauri already resolves; `cargo tree -p windows` to read it). `Win32_System_IO` covers `OVERLAPPED`/`CancelIoEx`/`GetOverlappedResult`; `Win32_Storage_FileSystem` covers `FILE_FLAG_OVERLAPPED` (both required by the sync-server overlapped accept — Step 7). `getrandom`: only add if `cargo tree` shows it absent AND the loopback fallback is later adopted — skip for now.

### Step 2 — `PeerIdentity` opaque type (touches Unix hot path)
In `platform/mod.rs`, define `pub enum PeerIdentity { #[cfg(unix)] Uid(u32), #[cfg(windows)] Sid(Box<[u8]>) }` (owned SID bytes on Windows) with `matches_current_process(&self) -> bool` (`#[cfg(unix)]`: `uid == libc::getuid()`; `#[cfg(windows)]`: `EqualSid(self_sid, process_owner_sid).is_ok()`) and a `fn display(&self) -> String` for the audit line (uid / `ConvertSidToStringSidW`). Change `PlatformListener::accept` (both unix + windows) to return `(PlatformStream, PeerIdentity)`. Update the two accept-loop callers (`hooks/server.rs`, `mcp/mod.rs`/daemon) to gate on `matches_current_process()` and log via `display()`. Update the comparison-branch unit test (`platform/mod.rs:840`) to exercise `PeerIdentity` instead of a bare `u32`.

### Step 3 — Windows security helpers module (`#[cfg(windows)]`)
A new `platform/win_sec.rs` (or an inline `#[cfg(windows)] mod`) holding, per research §2-5:
- `current_user_sid_string() -> io::Result<String>` (`OpenProcessToken` → `GetTokenInformation(TokenUser)` → `ConvertSidToStringSidW`).
- `owner_only_security_attributes(sid: &str) -> OwnedSecurityAttributes` (SDDL `D:P(A;;GA;;;{sid})` → `ConvertStringSecurityDescriptorToSecurityDescriptorW`; an RAII wrapper that `LocalFree`s on drop).
- `client_sid_of(server_handle) -> io::Result<Box<[u8]>>` (`ImpersonateNamedPipeClient` → **RAII RevertToSelf guard** → `OpenThreadToken(openasself=true)` → `GetTokenInformation(TokenUser)` → copy SID bytes out before the token/guard drop).
- `process_owner_sid() -> io::Result<Box<[u8]>>` and `sid_eq(a, b) -> bool` (`EqualSid().is_ok()`; verify the `Err`=not-equal semantics on the Windows machine — research §4 UNCONFIRMED).
- `verify_pipe_owner_is_current_user(client_handle) -> io::Result<()>` (`GetSecurityInfo(SE_KERNEL_OBJECT, OWNER_SECURITY_INFORMATION)` → `sid_eq` vs current user; note `GetSecurityInfo` returns `WIN32_ERROR`, check `== ERROR_SUCCESS` — research §5).
**Every `Impersonate*` path MUST construct the `RevertToSelf` guard before any `?`** (Decision 5 invariant).

### Step 4 — Windows endpoint-path helpers + cross-platform gui.pid dir
- Replace the `#[cfg(not(unix))]` branches of `hook_socket_path`/`mcp_socket_path`/`plan_review_fifo_path` (gate `#[cfg(windows)]`) to return `\\.\pipe\nergal-{current_user_sid_string}-{endpoint}` strings (hook / mcp / plan-{pid}).
- **gui.pid is used on Windows** (the liveness rule, Decision 6). Introduce a single resolver `gui_pid_dir() -> io::Result<PathBuf>` = `ipc_dir()` on Unix, `dirs::data_local_dir()/nergal` (created `0700`-equivalent, i.e. default per-user ACL) on Windows. **Both call sites must use it** (the Q3 wiring fix): the writer `lib.rs:651` (today `write_gui_pid(&ipc_dir()?)`, and `ipc_dir()` is `#[cfg(unix)]` → does not compile on Windows) and the reader `cli.rs:246` (today `read_gui_pid(fifo_path.parent())`, whose Windows parent is `\\.\pipe` — wrong) BOTH switch to `gui_pid_dir()`. `write_gui_pid`/`read_gui_pid` already take a dir and join `"gui.pid"`; `process_start_time` already uses `sysinfo` (cross-platform, `platform/mod.rs:450`); `check_gui_liveness` already has a `#[cfg(not(unix))]` arm (`:395-442`). So on Windows the per-tick liveness check fires exactly as on Unix once both call sites resolve the same dir.

### Step 5 — Named-pipe `PlatformListener`/`PlatformStream` body (`#[cfg(windows)]`)
Replace the `Unsupported` stubs (`platform/mod.rs:730-807`):
- `PlatformListener::bind(name)`: build `owner_only_security_attributes`, `ServerOptions::new().first_pipe_instance(true).create_with_security_attributes_raw(name, sa_ptr)` (map `PermissionDenied` → "squat detected, refusing" log + error), store the name; `LocalFree` the SD after create.
- `PlatformListener::accept()`: the multi-instance loop — `server.connect().await` (wait for client); **create the NEXT server instance BEFORE handing off** (Decision 9, shrink the `ERROR_PIPE_BUSY` window); the next instance uses `first_pipe_instance(false)` **but the SAME owner-only SA** (Decision 4 CRITICAL — a plain `create()` would give the default SD and re-open the foreign gap); extract `client_sid_of(server.as_raw_handle())` **fully synchronously, no `.await` between impersonate and revert** (Decision 5); build `PeerIdentity::Sid`; reject (close + rate-limited log) if `!peer.matches_current_process()`; else hand back `(PlatformStream(server), peer)`. Keep `max_instances` at the tokio default (unlimited).
- `PlatformStream`: wrap `NamedPipeServer` (accepted) / `NamedPipeClient` (connected); `AsyncRead`+`AsyncWrite` delegate to the inner; `connect(name)` = `ClientOptions::new().open(name)` with an `ERROR_PIPE_BUSY` retry loop (bounded `WaitNamedPipe`-style backoff, Decision 9), then `verify_pipe_owner_is_current_user` before returning.

### Step 6 — Migrate hook server + MCP daemon onto the seam
- `hooks/server.rs`: replace the raw `UnixListener::bind`/accept (gated `#[cfg(unix)]` by #1) with `PlatformListener::bind`/`accept`; the peer check becomes `PeerIdentity::matches_current_process()`. Un-gate (the seam handles both platforms now).
- `mcp/transport.rs` + `mcp/mod.rs`: replace `UnixSocketTransport` with `PlatformListener`; daemon accept loop uses `PeerIdentity`; `read_frame`/`write_frame` operate over `PlatformStream` unchanged. **Re-type the dispatch signatures (iprev #5)**: `handle_connection(stream: tokio::net::UnixStream, …)` (`mcp/mod.rs:619`) and `write_response(&mut tokio::net::UnixStream)` (`:670`) → `PlatformStream` (or generic `S: AsyncRead + AsyncWrite + Unpin`) — no raw `unix::net` at the call site. Remove/replace `DaemonContext.app_uid: u32` (`:82`), its use `uid != ctx.app_uid` (`:594`), and `test_ctx { app_uid: 0 }` (`:693`) with the `PeerIdentity::matches_current_process()` comparison. `lib.rs:692-714`: un-gate the daemon spawn (runs on Windows now via the seam); the raw `libc::getuid()` app_uid (`:700`) is no longer needed (the gate is `PeerIdentity`-based) — confirm no other ungated `libc::` re-enters the un-gated block (preserve the #1 invariant).

### Step 7 — Plan-review/ask-user gate (CLI-as-server; GUI connects-at-submit; Unix liveness rule — Decision 6)
- **New `#[cfg(windows)]` sync-server primitive** in `platform/mod.rs` — the overlapped-accept contract per research §7 (the named-pipe analog of the FIFO `O_NONBLOCK`+`poll(timeout)`):
  - `sync_listen(name)`: `CreateNamedPipeW` with `dwOpenMode = PIPE_ACCESS_DUPLEX | FILE_FLAG_OVERLAPPED | FILE_FLAG_FIRST_PIPE_INSTANCE`, `dwPipeMode = PIPE_TYPE_BYTE | PIPE_WAIT | PIPE_REJECT_REMOTE_CLIENTS`, `lpSecurityAttributes = Some(&owner_only_sa)`. **`FILE_FLAG_OVERLAPPED` is required** or `ConnectNamedPipe` blocks synchronously.
  - `accept_with_timeout(tick) -> io::Result<Option<(SyncPlatformStream, PeerIdentity)>>`: create a **manual-reset** event (`CreateEventW(None, true, false, None)`), put it in `OVERLAPPED.hEvent`, call `ConnectNamedPipe(handle, Some(&mut ol))` **once before the wait loop**. Return-value handling: `TRUE` → error (unexpected for overlapped); `FALSE`+`ERROR_IO_PENDING` → pending (wait); `FALSE`+`ERROR_PIPE_CONNECTED` → a client won the race = **success** (`SetEvent(ol.hEvent)`, proceed); `FALSE`+other → real error. Then loop `WaitForSingleObject(event, tick_ms)`: `WAIT_TIMEOUT` → return `None` (caller runs the liveness check and re-calls to wait again on the SAME pending op — do NOT re-issue `ConnectNamedPipe`); `WAIT_OBJECT_0` → connected, extract `client_sid_of` → `PeerIdentity`, return `Some`. On final give-up the caller `CancelIoEx(handle, Some(&ol))` + drains the completion before drop.
  - `SyncPlatformStream: std::io::Read + Write` over the connected handle. Reuse Step 3 helpers; the CLI has no tokio runtime (blocking Win32, not tokio).
- `hooks/cli.rs`: remove the broken `#[cfg(not(unix))]` `FifoGuard` placeholder (`:197-207`). On Windows the CLI hosts the gate via `sync_listen`, then loops `accept_with_timeout(≈1s)`: on `None`, run the **same GUI-liveness check as Unix** (`gui.pid` pid+start-time token → dead = safe deny) and enforce the `PLAN_REVIEW_WALL_CLOCK_SECS` backstop; on `Some((stream, peer))`, verify `peer.matches_current_process()` then read the decision. **`CreateNamedPipeW` MUST complete before the plan-review socket notification is sent** (mirror the Unix `mkfifo`@`:196`-before-socket-send@`:235` ordering). Unix FIFO path untouched.
- **GUI side** (`commands.rs` `submit_plan_decision`, `:424`): on Windows the GUI connects to the CLI gate pipe **at submit time** (single command invocation, no held state) via the sync/`PlatformStream` client — with `verify_pipe_owner_is_current_user` + `ERROR_PIPE_BUSY`/`ERROR_FILE_NOT_FOUND` retry — and writes the decision JSON, replacing `std::fs::write(decision_path, …)` (invalid against a pipe path). On Unix `std::fs::write` to the FIFO is unchanged. The GUI continues to write `gui.pid` (the liveness token) as today.
- `sync_connect` Windows body: replace `UnsupportedSyncStream` (from #1) with a synchronous named-pipe **client** connect returning a `Read + Write` handle, with `ERROR_PIPE_BUSY`/`ERROR_FILE_NOT_FOUND` retry (Decision 9) and `verify_pipe_owner_is_current_user`. Used by both the GUI submit-write and the CLI's non-gate fire-and-forget event sends.
- **Scope note (iprev round-3): only plan-review is a live blocking gate.** `ask_user` is non-blocking — `cli.rs:476-481` fires a socket and exits with no FIFO/pipe server, so there is no ask-user endpoint to host. `submit_ask_answer` (`commands.rs:362`) is a second `std::fs::write(decision_path)` site; on Windows that path is a pipe name no server hosts, so the write fails harmlessly (the actual answer returns through CC's native TUI over the PTY, as today — `ask_user` is purely an attention notifier). Do NOT stand up an ask-user pipe server on Windows; only `submit_plan_decision` needs the connect-and-write (the CLI-as-server sync primitive is plan-review-only).

### Step 8 — IPC observability
Ensure every Windows rejection (squat-detected bind failure, foreign-SID accept rejection, owner-mismatch client refusal) emits a logged audit line via the existing `RejectionRateLimit` (`platform/mod.rs:509`), keyed by the SID string, so the Windows boundary is as observable + DoS-resistant as the Unix one.

## Per-step risk + mitigation

- **Step 3/5 (RevertToSelf invariant)** — the one CRITICAL correctness point: a missed revert on an error path is a privilege hole. Mitigation: RAII guard struct, constructed before any fallible call; iprev confirms.
- **Step 2 (PeerIdentity on the Unix hot path)** — a regression here drops the only enforced Unix access control. Mitigation: the comparison-branch CI test (spec "No Linux regression") + full Linux machine-check.
- **Step 5/3 (Win32 return-type gotchas)** — `EqualSid` (Err=not-equal) / `GetSecurityInfo` (WIN32_ERROR). Mitigation: research §4/§5 exact checks; verify on the Windows machine; `windows-sys` raw bindings as the fallback.
- **No local validation on Linux** for the named-pipe runtime — `windows-latest` CI compiles + unit-tests the comparison branch; the real two-user SID-rejection harness is UNVERIFIED-pending until the user runs it on the Windows machine.
