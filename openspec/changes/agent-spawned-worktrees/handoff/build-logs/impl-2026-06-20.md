## Session impl-2026-06-20 · agent-spawned-worktrees (#3 MCP)

### What shipped
Full implementation (phases 1–5 + tests). Core: `src-tauri/src/mcp/worktree_sessions.rs`.

- **Request tools** (sync, non-blocking): `create_worktree_session` / `get_worktree_request_status` / `cancel_worktree_request` wired in `mcp/mod.rs` dispatch + `tool_definitions`. Two-structure gate (`GateInner { pending, ledger }`) under one `Mutex` on a new `DaemonContext.worktree_gate` field, shared with the Tauri commands as managed state.
- **Human gate** (Tauri, GUI-only): `list_worktree_requests` / `approve_worktree_request` / `deny_worktree_request` + `agent_worktrees_set_enabled`. - **Outcome delivery**: `notify_outcome` (idle-only PTY wake + `worktree:resolved` event); poll fallback. Timeout sweeper `run_worktree_request_sweeper` (atomic purge + ledger GC) spawned in `lib.rs`.
- **Config**: `AgentWorktreesConfig` (default off) + `BACKEND_OWNED_CONFIG_KEYS` + Settings toggle in `McpSection.tsx`.

### Decisions + divergences vs the redaction
- **No Stop-`additionalContext` fast-path** for outcome delivery: that path is DB-queue-driven (cross-session 4.3, deferred). The idle PTY wake + poll fallback are the delivery — honest and sufficient (a working requester polls).
- **Slug collision check is defensive**: `derive_worktree_slug` appends a fresh `ts`, so a real collision is effectively unreachable; the check is kept for parity + spec conformance.
