## 4. Poller + atomic reconcile

- [ ] 4.1 `linear/poller.rs`: interval task (configurable cadence `linear_poll_interval_secs`, default 45s, floor 10s). **Capture `key_generation` at cycle start** (iprev r4 #1). Resolve `viewer`; a failed/empty resolve **skips the cycle, never wipes** (iprev r3 #1); a successful resolve seeds `viewer_id` into the null slot left by `set_key` (no "non-null differs → wipe" branch — the epoch handles swaps, iprev r4 #2). Capture **one `cycle_now`** driving both the server filter and the candidate predicate (iprev #10). Per cycle, scoped to selected teams: fetch teams+states+labels, then the **bounded issue scope** (design D4) = union of (1) `issues_page(team_ids=selected, updated_after=cycle_now-window)` cursor-paginated, (2) `viewer_assigned_issues(team_ids=selected)`, (3) **delta-only** `issues_by_id(mirror_mine − ids(1) − ids(2))` chunked+paginated, full-relation (iprev r2 #2/r4 #3). Track whether **every branch (incl. every set-3 chunk) reached `hasNextPage==false`** (the `complete` flag, iprev r3 #3). **Then** commit in one SQLite transaction that **re-reads `key_generation` and aborts/discards the commit if it changed** since cycle start (iprev r4 #1 — a swap mid-cycle cannot re-populate the wiped mirror). Use `BEGIN IMMEDIATE` and make the generation re-read the **first** statement of the write transaction, so the snapshot cannot predate a concurrent `set_key` (iprev r5 #1).
- [ ] 4.2 Ordered upsert within the transaction: `teams → workflow-states → labels → projects → cycles → users → issues → issue-labels → comments`. An issue whose non-null `state_id`/`team_id` is unknown synthesizes a placeholder row flagged `synthetic` (+ logs), never an FK panic; a non-null `parent_id` to an out-of-scope parent stays a tolerant root (plain column). Set/clear `was_viewer_assigned` from the fetched assignee. Label defs upsert-only.
- [ ] 4.3 **Completeness-gated tombstoning + eviction** (iprev #1/#4/#15/#17, r2 #1/#6, r3 #2/#4): tombstone only if `complete`. Candidate = belongs to a selected team AND `updated_at > cycle_now - window` AND absent from the cycle's **global** fetched-id set (moved-team survives) AND not `synthetic` AND not `was_viewer_assigned`. **Set-3 outcome (presence-keyed; `issues_by_id` returns NO 404 — a gone id is simply absent):** returned still-mine + team-still-selected → **retain flag, full upsert, no evict** (4th case, set-2-was-incomplete — iprev r4 #3); returned with `assignee != viewer` → clear `was_viewer_assigned`; returned with `team_id` not in selected → clear flag + evict (no un-resolvable placeholder); **absent from a COMPLETE set-3** → clear flag + evict (the only deleted-while-mine path); **absent from an INCOMPLETE set-3 → retain unchanged** (absence = "didn't reach it", acting on it would delete real issues). Then the **eviction pass** (same transaction, after upserts+tombstones — iprev r2 #6): hard-delete issues with `updated_at <= cycle_now - window` AND not `was_viewer_assigned` (childless first); mine-flag clears feeding eviction come **only from a complete set-3** (iprev r3 #4). Reappearance resets `stale=0`. If `!complete`: upserts only, no tombstones, no absence-based set-3 eviction; log `incomplete fetch — tombstoning skipped`.
- [ ] 4.4 Sub-issue tree built in app from `parent_id` (from the flat `parent` relation), tolerant of a dangling/out-of-scope parent (render as root). No separate detail call drives the reconcile tree.
- [ ] 4.5 **First-sync baseline + viewer-change reset** (iprev #8/#14, r2 #3): while `baseline_done=0`, seed silently — zero notifications. Set `baseline_done=1` only after a **complete** sync over a **non-empty** team selection. The viewer-change reset is driven by the **per-cycle viewer resolve** in 4.1 (not only `linear_validate_key`); `linear_set_key` also clears stored `viewer_id` so the next cycle re-resolves. Only post-baseline new assignments to the viewer notify; coalesce a bulk re-assignment into one "N new issues assigned" notification.
- [ ] 4.6 Lazy heavy sub-data: comments fetched on detail-open or when `updatedAt` advanced; not every poll.
- [ ] 4.7 Emit `linear:changed` after each committed reconcile; `linear:sync-status` on state changes; `linear:assigned` (coalesced names) post-baseline. **Assignment notifications fire only AFTER the commit + epoch check** (not off the pre-commit fetch diff), so an epoch-aborted old-account cycle emits no stale `notify-send` (iprev r5 #2). Stale GC past a retention window, childless-only; `gc_unreferenced_labels` each cycle.
- [ ] 4.8 Tests: first sync of a pre-assigned workspace → zero notifications; new issue; **reassigned-away issue detected via set-3 delta even with no `updatedAt` bump and out of window** → `was_viewer_assigned` cleared, assigned-to-me filter hides it, NOT tombstoned; **set 3 fetches only the delta** (a still-assigned issue already in set 2 is excluded from the by-id batch — iprev r2 #2); **mine-issue moved to an UNSELECTED team → flag cleared + evicted, no perpetual placeholder** (iprev r2 #1); **interrupted/rate-limited pagination → zero tombstones** (iprev #1); **aged-out non-mine issue → evicted, not lingering** (iprev #4); **issue moved A→B between selected teams → survives, team_id=B** (iprev #15); **synthetic placeholder → not tombstoned** (iprev #17); **viewer-change via per-cycle resolve → wipe + silent re-baseline** (iprev #8/r2 #3); **zero-team sync → no baseline set** (iprev #14); **viewer-resolve FAILURE (rate-limit/network) → mirror NOT wiped, cycle skipped** (iprev r3 #1); **deleted-while-mine surfaces as by-id ABSENCE → cleared+evicted only on a COMPLETE set-3** (iprev r3 #2); **INCOMPLETE set-3 → mine-flags retained, no eviction** (iprev r3 #2/#4); **delta larger than one by-id page → all chunks re-verified** (iprev r3 #3); **key swapped mid-cycle → the in-flight cycle's commit is discarded, old-account data not written** (iprev r4 #1); **set-3 returned-still-mine-still-selected → flag retained, labels NOT stripped** (iprev r4 #3); **`set_key` clears team selection + bumps generation** (iprev r4 #4); reappearance → un-tombstoned; new post-baseline assignment → one notification; bulk → coalesced; out-of-scope-parent placeholder path; atomic-commit incl. eviction in-txn (no torn/half-evicted read — r2 #6); viewer-assigned issue older than the window still synced via sets 2/3.
